Contribution Guidelines

Thank you for considering contributing to this repository!

Ways to Contribute

  1. Add Tutorials - Create new tutorials or improve existing ones
  2. Add Scripts - Contribute useful example scripts
  3. Fix Issues - Report and fix bugs
  4. Documentation - Improve README and guides
  5. Examples - Add real-world use cases

Process

  1. Fork the repository
  2. Create a branch: git checkout -b feature/my-feature
  3. Make changes
  4. Test your code
  5. Commit: git commit -m "Add feature: description"
  6. Push: git push origin feature/my-feature
  7. Open Pull Request

Guidelines

Tutorials

  • Organize by difficulty level (beginner/intermediate/advanced)
  • Include clear explanations and examples
  • Add practice exercises
  • Link to official documentation

Scripts

  • Include docstrings
  • Handle errors gracefully
  • Make executable and testable
  • Add usage examples

General

  • Use clear, descriptive commit messages
  • Test code before submitting
  • Follow Python PEP 8 style guide
  • Update README if adding new sections

Questions?

  • Check existing issues first
  • Create a new issue for bugs or suggestions
  • Start a discussion for questions

Code of Conduct

  • Be respectful and inclusive
  • Provide constructive feedback
  • Help others learn
  • No harassment or discrimination

License

By contributing, you agree your contributions are licensed under the same license as the project.

Thank you for contributing! 🎉
